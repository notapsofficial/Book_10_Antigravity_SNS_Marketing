# 第三章：最初の通信（Hello World）

いよいよ、あなたのコードが世界に向かって声を上げる瞬間です。

この章では、最小限のコードで「Xに投稿する」という機能を実装します。これは、遠大なパイプラインの全ての基礎となる部分です。

## 1. AIにコードを書かせる

コードを一行一行書く必要はありません。
「何をしたいか」を正確に伝えるだけで、AIが最適なコードを生成してくれます。

フォルダの中に `post_tweet.py` というファイルを作るようにAIに依頼しましょう。

### プロンプト例：投稿スクリプトの作成

```markdown
以下の要件で、X（Twitter）にテキストを投稿するPythonスクリプトを作成してください。

### 要件
1. ファイル名: post_tweet.py
2. ライブラリ: tweepy (API v2 Clientを使用)
3. 認証: .envファイルからAPI Key, API Secret, Access Token, Access Secretを読み込むこと
4. 処理内容:
   - Clientを作成し、認証する
   - "This is a test post from Python! #AntigravityOS" というテキストを投稿する
   - 投稿に成功したら、投稿IDを表示する
   - エラー時（認証エラーや権限エラー）は、分かりやすい日本語のエラーメッセージを表示して終了する
```

AIが生成したコードをコピーし、`post_tweet.py` ファイルに貼り付けて保存してください。

## 2. 実行する

ターミナルで以下のコマンドを打ち込み、Enterキーを押します。

```bash
python post_tweet.py
```

うまくいけば、`投稿成功！ ID: 123456789...` という表示が出ます。

自分のスマホでXを開き、投稿されているか確認してください。その瞬間、あなたはただのユーザーから「API使い」に進化しました。

## 3. よくあるエラー（詰まりポイント3）

### "403 Forbidden"
最も多いエラーです。99%の原因は、第一章で警告した「Read and Write」の設定漏れ、もしくは設定後にTokenを再発行（Regenerate）していないことです。

Dashboardに戻り、設定を確認し、Tokenを再発行して、`.env`を書き換えてください。

### "403 Forbidden - Duplicate Content"
2回目に同じコードを実行すると、「あなたは同じ内容をすでに投稿しています」と怒られます。Xはbotによるスパムを防ぐため、同じ内容の連投を禁止しています。

テストするときは、`text = "Test 2"` のように中身を毎回変える必要があります。

一回だけの投稿なら、手でやった方が早いです。
しかし、私たちは「自動化」を目指しています。
