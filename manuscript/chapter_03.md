# 第3章：技術力で解決しようとした（そして沼にハマった）

## 1. 最強の自動化システムを構築した

コピペ運用に限界を感じた私は、AIを使いこなす「現代の作り手」としてのプライドを取り戻すべく、技術的な解決を試みました。
「Antigravity」に出会う前の話です。当時の私は、**Cursor** というAIエディタの力を借りて、一見完璧な自動化システムを構築しました。

**【当時のアーキテクチャ】**
1.  **Cursor** でPythonコードを実装。プロンプトエンジニアリングも自分で行う。
2.  **OpenAI API** を呼び出し、投稿案を生成させる。
3.  **GitHub Actions** の定期実行（Cron）で、毎日定時にトリガーを引く。
4.  炎上リスクを避けるため、生成されたテキストを **Discord** のWebhookに投げる。
5.  私がDiscord上で内容を確認し、OKなら「承認ボタン」を押す。
6.  承認されたものだけが、API経由で **X (Twitter)** に投稿される。

完璧な布陣に見えました。
これなら、IDEを開く必要も、ブラウザを行き来する必要もありません。
通知が来たら、Discordでポチッと押すだけ。「Human-in-the-loop」を取り入れた、理想的なモダン・オプスです。

## 2. Discord上の「レビューガチャ」

しかし、運用を始めてすぐに、このシステムには致命的な欠陥があることに気づきました。
それは、AI（OpenAI）の生成品質における**「ガチャ要素」**です。

朝、Discordに通知が来ます。
「今日の投稿案です：『プログラミングは楽しいですね！今日も一日頑張りましょう！』」

……違う。
私のターゲットは個人開発者であり、そんな無難な挨拶を求めているわけではない。
私は「Reject（却下）」ボタンを押します。システムが再生成し、また通知が来ます。

「エンジニアあるある：セミコロンを忘れてエラーが出ることってありますよね？」

……それも違う。あまりに手垢のついたネタすぎる。
再び「Reject」。

結局、私はDiscord上で何度も「再生成」を繰り返し、納得いく案が出るまでスマホをポチポチし続けることになりました。
これでは、自分で書くのと手間が変わらないどころか、「他人の書いた微妙な文章を添削し続ける」という、さらにストレスの溜まる作業に変わっただけでした。

## 3. 「ちゃんと投稿されたか」症候群

さらに厄介だったのが、承認後の挙動です。
Discordで「Approve」ボタンを押し、Botが「投稿しました」と返しました。
本来なら、ここでスマホを閉じればいいはずです。

しかし、私は不安で **X アプリを開いていました。**
「本当に投稿されたかな？」
「画像は崩れてないかな？」
「変な改行が入ってないかな？」

結局また、あの「時間を溶かす場所」に戻ってきているのです。
そしてアプリを開いたついでに、通知タブを見て、おすすめフィードを見て……気がつけば30分が経過しています。

## 4. 構造的欠陥への気づき

この「Discord Bot運用」の失敗で、私は完全に理解しました。
問題は、ツールの有無でも、自動化のレベルでもありませんでした。

*   **レビューコスト**: AIの精度が私の「お気持ち」とズレている限り、承認フローは機能しない。
*   **確認コスト**: プラットフォーム（X）への不信感や不安がある限り、結局アプリを開いてしまう。

私が作ったのは「自動化システム」ではなく、「Discordという経由地点を増やしただけの、手間のかかるルーブ・ゴールドバーグ・マシン（ピタゴラ装置）」だったのです。

「頑張ってもダメ、AIを使ってもダメ、自動化コードを書いてもダメ」

完全に手詰まりでした。
しかし、この「全通り試して全滅した」という事実こそが、私に最後の選択肢を提示してくれました。
それが、ツールではなく**「概念（コンセプト）ごと丸投げする」**という、Antigravityのアプローチでした。
