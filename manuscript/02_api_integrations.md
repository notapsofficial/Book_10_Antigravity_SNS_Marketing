# 第2章：触手を伸ばす - API連携の壁

ダッシュボードという「目」を手に入れたら、次は世界に干渉するための「手（API）」が必要です。
X (Twitter)、Instagram、YouTube、Pinterest...。これらを自由に操作できれば、あなたはネットの神になれます。
しかし、そこには巨大な壁が立ちはだかっています。**「公式ドキュメント（API仕様書）」**という壁です。

## 技術書は「読まない」が正解

各プラットフォームの公式ドキュメントは、難解な英語で書かれ、しかも頻繁に変更されます。人間が読むようには作られていません。
だから、私たちは読みません。AIに読ませるのです。

ここでの最強ツールは、AIの**「Webブラウジング機能」**です。

### ケーススタディ：X (Twitter) API v2 の迷宮

私がXへの自動投稿を実装しようとした時、ネット上の解説記事はほとんどが古く（v1.1）、役に立ちませんでした。
そこで私は、最新の公式ドキュメントのURL（例えば `developer.twitter.com` の該当ページ）を探し出し、Antigravityにこう投げかけました。

> **私**: 「このURLを読んで、Pythonで画像付きツイートをするスクリプトを書いて。OAuth 1.0a認証を使うこと。[URL]」

AIは数十秒で数千行のドキュメントを読み込み、「はい、これがコードです」と返してきました。
私は認証キー（API Key）を入れるだけ。
「え、これでいいの？」と拍子抜けするほどあっさり通りました。

### 【Tips】URLを「エサ」として与える
やり方がわからなければ、Google検索上位の解説記事URLを片っ端からコピーして、
「この記事とこの記事を読んで、私のやりたいことに応用して」と指示します。
**人間が情報を消化する必要はありません。情報の「場所」だけを指示し、消化と出力はAIにさせる。**
これが、AI時代の学習法です。

## 絶望のPinterest審査落ち

しかし、コードが正しくても通れない壁があります。
PinterestのAPI申請です。ここだけは、人間による厳しい「審査」がありました。

申請を出して数日後、悲しいメールが届きました。
「Rejected（却下）」
理由は、「アプリがどう使われるかわからないから、デモ動画を見せろ」とのこと。

「デモ動画？ まだアプリなんて完成してないよ！ スクリプトしかないよ！」

私はパニックになりかけましたが、すぐにAntigravityに相談しました。

> **私**: 「審査に落ちた…。『デモ動画を送れ』って言われたんだけど、どうすればいい？ 画面のないPythonスクリプトだよ？」

Antigravityの回答は冷静かつ、目からウロコでした。

> **AI**: 「大丈夫です、よくあることです。以下の手順で動画を作りましょう。
> 1. VS Codeの画面を録画します。
> 2. スクリプトを実行するコマンドを打ちます。
> 3. ログが流れて『Success』と出る様子を見せます。
> 4. 最後にPinterestのブラウザ画面を開き、実際にピンが投稿されたところを見せます。
> これで『動作確認』になります。」

さらにAIは、**「審査担当者に送る申し開きメールの英文」**まで書いてくれました。

> "Dear Review Team, I have attached a video demonstrating the core functionality..."

言われた通りに画面をキャプチャし、AIが書いた英文を添えて再提出。
翌日、「Approved（承認）」のメールが届きました。
AIがいなければ、確実にここで挫折していたでしょう。

## 403 Forbidden の恐怖

APIを使っていると、突然「403 Forbidden（アクセス権がない）」というエラーが出ることがあります。
多くの初心者はここで「キーの設定間違えたかな？」と悩みますが、原因は様々です。
*   プランの制限（無料枠を超えた）
*   書き込み権限（Read/Write）の設定ミス
*   エンドポイントのURL変更

ここでも鉄則は**「ログの丸投げ」**です。
私は以前、YouTube APIで動画をアップロードしようとしてエラーが出ました。

> **私**: （エラーログ貼り付け）「昨日まで動いてたのに急にダメになった。なんで？」
> **AI**: 「ログを見ると `quotaExceeded` とありますね。YouTubeの1日のアップロード割り当て量（Quota）を超えています。これ以上のテストは明日のリセット時間を待つか、もっと効率的にAPIを叩く必要があります。」

「悩む」時間をゼロにする。
これが、一人で4つのプラットフォームを管理し続けられる唯一の秘訣です。
