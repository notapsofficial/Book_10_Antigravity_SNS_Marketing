# Part 5｜詰まらないための実務フロー（API・UI・セキュリティ）

## 1. エラーは「失敗」ではなく「信号」だ

### 400番台：あなたが悪い
システムを動かせば、必ず赤い文字（エラー）が出る。
初心者はそこでパニックになるが、システムオーナーは冷静にログを読む。

*   **400 Bad Request**: データの形式が間違っている。「画像がデカすぎる」「文字数が140字を超えている」。これはあなたのコードが悪い。直せば動く。
*   **401 Unauthorized**: 鍵が違う。「.envのキー間違ってない？」「コピペミス？」
*   **403 Forbidden**: 権限がない。「Read Onlyになってない？」「APIプランの支払いは有効？」
*   **429 Too Many Requests**: 落ち着け。リクエストしすぎだ。15分待て。

「動かない！」とAIに叫ぶ前に、ステータスコードを見ろ。
400番台なら、あなたが直せる。

### 500番台：世界が悪い
*   **500 Internal Server Error**: Xのサーバーが燃えている。
*   **502 Bad Gateway**: 中継サーバーがおかしい。
*   **503 Service Unavailable**: メンテ中か、過負荷。

これらは**「寝て待て」**という合図だ。
あなたがどれだけコードを弄っても直らない。
PCを閉じて、散歩に行こう。それが正しい対処法だ。

## 2. AIへの「目の提供」：スクリーンショット

### 言葉で説明するな、見せろ
Developer Portal の設定画面などで詰まった時、状況を言葉で説明するのは非効率だ。
「青いボタンの下にある設定項目がグレーアウトしていて...」
そんな説明をチャット欄に打つ暇があったら、**Cmd + Shift + 4** を押せ。

最新のマルチモーダルAI（GPT-4o, Atlas, Geminiなど）は、画像認識能力が極めて高い。
スクリーンショットを貼り付け、「これで権限エラーが出る。どこを直すべき？」と聞けば、
「右上の『Edit』ボタンを押して、『App permissions』をトグルしてください」と、座標単位で正解を教えてくれる。

### [重要] 黒塗りのマナー
ただし、**API Key や Access Token が映っている画面**をそのままアップロードしてはいけない。
AIの学習データに残るリスクがあるし、チャット履歴から漏れる可能性もゼロではない。
Macのプレビューでも、ペイントでもいい。
**重要な文字列は必ず黒塗り（Masking）してからアップロードせよ。**
これはデジタルタトゥーを入れないための、最低限の嗜みだ。

## 3. Deep Research の使いどころ

### 2026年の仕様は2024年のAIにはわからない
通常のチャットAIは、学習データのカットオフ（知識の期限）がある。
「X API v2 の最新仕様」を聞いても、古い情報を堂々と嘘つくことがある。

ここで使うのが **Deep Research（深層検索）** 機能だ。
これはAIが自らGoogle検索を行い、最新のドキュメントを読みに行ってくれる機能だ。

### 使うべき場面 vs 使わなくていい場面

*   **使うべき (Use Deep Research)**:
    *   「2026年1月現在の X API Basic Plan の料金と制限は？」
    *   「Twitter Developer Portal のUIが変更されたらしいが、新しい作成フローは？」
    *   「競合の〇〇というサービスが最近始めた新機能について知りたい」
    *   **事実確認（Fact Check）が必要なすべての場合。**

*   **使わなくていい (Just Chat)**:
    *   「Pythonでリストをソートするコードを書いて」
    *   「このエラーログの意味を教えて」
    *   「もっと皮肉の効いた言い回しを考えて」
    *   **論理演算や創作に関しては、検索不要のモデルの方が速くて優秀だ。**

この使い分けができるようになれば、あなたの「検索コスト」は劇的に下がる。
