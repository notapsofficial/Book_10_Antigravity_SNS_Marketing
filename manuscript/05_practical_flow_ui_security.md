# Part 5｜運用のための実務ガイド（トラブルシューティングと安全策）

## 第1章：エラーとの付き合い方

### 1-1. エラーは信号に過ぎない

システムを動かし始めると、予期せぬエラーに遭遇することがあります。
見慣れない赤い文字が表示されると、少し驚くかもしれませんが、心配はいりません。
エラーログは、システムがあなたに送ってきている**「状況を知らせるメッセージ」**です。

HTTPステータスコードの意味を知っているだけで、落ち着いて対処できるようになります。

### 1-2. 400番台：設定を見直すサイン (Client Error)

400番台のエラーが出た場合、多くは設定や入力内容の調整で解決します。

*   **400 Bad Request**:
    *   **状況**: リクエストの内容が形式に合っていない場合です。
    *   **ヒント**: 文字数が超過していないか、画像サイズが大きすぎないかを確認してみましょう。
*   **401 Unauthorized**:
    *   **状況**: 認証が上手くいっていません。
    *   **ヒント**: APIキーのコピペミスや、環境変数が正しく読み込まれていない可能性があります。`.env` ファイルを落ち着いて見直してみましょう。
*   **403 Forbidden**:
    *   **状況**: 権限が足りない状態です。
    *   **ヒント**: 開発者ポータルで `Read Only` になっていないか、プランの状態は有効かを確認します。
*   **429 Too Many Requests**:
    *   **状況**: 一時的にアクセスしすぎています。
    *   **ヒント**: 少し休憩のサインです。15分ほど待ってから再試行すれば、通常は解消します。

これらは「失敗」ではありません。単なる「設定調整の合図」です。

### 1-3. 500番台：待つことが正解 (Server Error)

*   **500 Internal Server Error**
*   **503 Service Unavailable**

これらはX（プラットフォーム）側で何か起きているサインです。
こちらのコードの問題ではないため、焦って修正する必要はありません。
コーヒーでも飲んで、復旧を待つのが最も適切な対応です。

#### Tips: 「初心者にわかるように」という魔法の言葉
エラーの意味がわからず不安な時は、エラー文をそのままコピーして、こう付け加えてみてください。
**「このエラーの原因と対処法を、非エンジニアにもわかるように、専門用語を使わずに解説してください」**
AIは、難解な技術用語を避けて、「今はXのサーバーが混み合っているだけなので、何もせずに待っていれば大丈夫ですよ」といった、安心できる言葉に翻訳してくれます。
恐怖心を取り除くために、AIの翻訳機能を使いましょう。

## 第2章：AIへの「目の提供」：スクリーンショット活用

### 2-1. 画像で伝えるとスムーズな場合も

開発者ポータルの設定など、画面操作で迷ったときは、AIの画像認識機能を頼るのも一つの手です。
状況を文章で説明しようとすると大変ですが、スクリーンショットを見てもらうことで、スムーズに解決策が見つかることがあります。

「設定画面でエラーが出るのですが、どこを確認すれば良いですか？」
と画像を添えて尋ねれば、多くのAIは的確なアドバイスを返してくれます。

### 2-2. 安全のためのマナー（黒塗り）

ただし、画像をアップロードする際には、一つだけ大切な安全ルールがあります。
**「重要な情報は隠す（黒塗りする）」**ということです。

APIキーやアクセストークン、あるいは個人のメールアドレスなどが映り込んでいる場合は、必ず隠してからアップロードしましょう。
Macのプレビュー機能や、ペイントツールなどで簡単に塗りつぶせます。

これは、意図しない情報の拡散を防ぐための、我々エンジニアにとってのシートベルトのようなものです。
習慣にしてしまえば、安心してAIを活用できます。

## 第3章：Deep Research の活用シーン

### 3-1. 最新情報は検索機能で補完する

ご存知の通り、AIモデルの学習データには期間的な区切り（知識の期限）があります。
そのため、プラットフォームの最新の仕様や変更点については、AIが古い情報を参照してしまうことがあります。

### 3-2. 検索機能（Browsing）の使い分け

そのような時は、AIの **Web Browsing（検索）機能** を有効活用しましょう。

#### 検索機能が役立つ場面
*   「現在の X API Basic Plan の料金体系を知りたい」
*   「最近発表されたAPIの変更点について確認したい」
*   「他社サービスの最新機能を調べたい」

このように「最新の事実」を確認したい場合は、検索機能をオンにします。
逆に、コードのロジックや文章の作成など、普遍的なタスクについては、通常のチャットモードの方がレスポンスも速く快適です。
目的に応じて道具を使い分けることで、より効率的に作業を進められます。
