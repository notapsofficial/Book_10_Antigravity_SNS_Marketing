# 第7章：SNSごとのAPI事情と「共通プロトコル」

## 1. 結局、全部やってみた

「Xで行こう」と決める前、正直に言うと、私は**全部試しました**。
エンジニアとしての好奇心（あるいは悪癖）が騒ぎ出し、気になったプラットフォーム全てのDeveloper Portalの扉を叩いたのです。

*   **X (Twitter)**
*   **TikTok**
*   **LinkedIn**
*   **Instagram / Facebook**
*   **Threads**

これら全てに対して、開発者登録を行い、APIキーを取得し、Antigravityと接続を試みました。
その結果、ある一つの重要な事実に気づきました。
プラットフォームごとに用語や画面は違えど、やるべきことの**「本質的な構造」は全く同じ**だったのです。

## 2. API接続の「共通プロトコル」

どのSNSであっても、私たちが通るべき道は以下の4ステップに集約されます。

1.  **Developer Accountの開設**
    *   自分の通常アカウントに紐付ける形で、開発者としての登録を行う。
2.  **API Key / Token の発行**
    *   「アプリ」を作成し、認証情報（IDとパスワードのようなもの）を取得する。
    *   ここで重要なのは、**「投稿権限（Write Permission）」**を確実に付与することです。
3.  **Antigravityへの委譲（.env）**
    *   取得したキーを、生のままプロンプトに貼るのではなく、環境変数 `.env` に格納してAntigravityに渡す。
4.  **接続テストと発射**
    *   「Hello World」を投稿してみる。

XだろうがTikTokだろうが、この流れは変わりません。
つまり、一つのプラットフォームでこの「作法」を体得してしまえば、他のSNSに展開するのは容易だということです。

## 3. 「手順書」は書かない

本書では、各プラットフォームの具体的な画面操作（「右上のボタンを押して...」など）については解説しません。
なぜなら、**その情報は来月には古くなっているから**です。

XのDeveloper Portalは頻繁にUIが変わります。TikTokの審査フローも常に変動しています。
検索して出てくる半年などの「図解入り手順記事」が、全く役に立たなかった経験は誰にでもあるはずです。

しかし、**「AI（Antigravity）と一緒に突破する方法」**だけは不変です。

画面が変わっていても、「今はこういう画面が表示されている。私の目的はAPIキーの取得だ。どこを見ればいい？」とAntigravityに聞けばいいのです。
仕様変更に追従するのは人間の仕事ではなく、AIの仕事です。

## 4. この本の強み

この本が提供するのは、賞味期限の短い「操作手順」ではありません。
どんなに仕様が変わろうとも、新しいSNSが登場しようとも対応できる、**「構造的なアプローチ」**です。

*   仕様が変わったら、エラーログをAIに食わせる。
*   画面が変わったら、スクショをAIに食わせる。
*   常に「共通プロトコル」を意識して、迷子にならないようにする。
*   [ ] 常に「共通プロトコル」を意識して、迷子にならないようにする。

**「戦う前に戦場を調べ尽くした。それが戦略の本質です」**

次章以降では、私が最初に攻略した「X」を題材にしますが、そこで語られる**「キー管理の重要性」**や**「デバッグの作法」**は、あなたがTikTokやLinkedIn挑む際にもそのまま使える武器になるはずです。
