# 第6章：無料でどこまで戦えるか

## 1. 「月額100ドル」の壁

Antigravityに導かれるまま、「X Developer Platform」のページを開いた私は、その価格表を見て凍りつきました。
まともにボットを作ろうとすると推奨される **"Basic Tier"** の価格は、**月額100ドル（約1万5千円）**。

「さすがに、まだ収益化もしていない個人アプリの広報に、毎月1万5千円は出せない……」

第6章のタイトルを「投資する決断」にしようかとも思いましたが、正直に言います。私にはその勇気がありませんでした。
もし全く効果がなかったら？ アカウントがBANされたら？
リスクが高すぎます。私はAntigravityに「待った」をかけました。

「有料は無理だ。無料（Free Tier）でなんとかならないか？」

## 2. Free Tier のスペックを再確認作戦

Antigravityは即座に、2025年12月時点での **Free Tier** の仕様を再調査してくれました。

> 「XのAPI仕様書（v2）を詳しく確認しました。
> Free Tierには厳しい制限がありますが、あなたの用途（自分自身の広報アカウントとしての運用）であれば、実は十分かもしれません。
>
> *   **投稿（Write）**: 月間 1,500件 まで可能。
> *   **読み込み（Read）**: 基本的に不可（自分の投稿の削除などは一部可）。
> *   **エンドポイント**: v2 のみ利用可能。
>
> 月間1,500件ということは、**1日あたり約50件**です。
> Botとして大量にリプライを返したり、全タイムラインを取得したりすることはできませんが、『あなたが1日に数回、開発日誌を投稿する』だけであれば、この制限に引っかかることはまずありません」

## 3. 「書くだけ」なら無料で行ける

この分析は目から鱗でした。
「API＝有料化した」というニュースばかりが目立っていましたが、詳細を見れば、「ごく普通の頻度で投稿する」だけのBotなら、無料で許容されていたのです。

むしろ、私の目的は「リプライ自動返信」でも「データ分析」でもありません。
ただ、「私の代わりに投稿してもらうこと」です。
読み込み（Read）制限があるため、エゴサーチや自動いいねはできませんが、それは「やらない理由（SNSを見ない理由）」としてむしろ好都合です。

「読み込めない。だから、見なくて済む」
「書ける数は決まっている。だから、無駄撃ちしない」

まさに、私が求めていた「アンチ・ドゥームスクロール（無限スクロール防止）」の仕様そのものでした。

## 4. 戦略の修正

私は戦略を修正しました。
月額100ドルを払ってフル機能を手に入れるのではなく、**「無料の制約の中で、賢く運用する」**道を選びました。

*   **投稿頻度**: 1日 3〜5ツイート（余裕でFree枠内）。
*   **機能**: こちらからの発信のみ（一方通行）。
*   **リスク**: お金はかからない。もしダメでも、失うものは時間だけ。

「Antigravity、Free Tierで行くぞ。
リプライへの反応とか複雑なことはしなくていい。純粋な『アウトプットマシン』を作ろう」

「了解しました。Free Tierならではの制限（v2エンドポイント必須など）がありますので、コード生成時はそこを考慮します」

こうして私は、クレジットカードを取り出すことなく、無料でDeveloper Portalの「Sign Up」ボタンをクリックしました。
ミニマムで、リーンな機能開発のスタートです。

## 到達状態 (Check Point)

*   [ ] Basic Tier（$100/mo）は高すぎると判断し、Free Tierを選んだ。
*   [ ] Free Tierの制限（月1500件投稿）が、個人の運用には十分だと理解した。
*   [ ] 「Readできない」ことを、「SNSを見ないで済むメリット」と捉え直した。

次章、他のプラットフォームには目もくれず、XのFree Tier一本に絞った理由を、「テキスト」という観点から深掘りします。
