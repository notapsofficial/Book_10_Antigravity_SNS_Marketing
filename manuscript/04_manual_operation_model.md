# Part 4｜手動×Antigravityで“事故らない”運用

## 1. ワークフロー：部長と新入社員の関係

### いきなり自動化＝事故の元
APIがつながった。キーも設定した。
「よし、cronで毎日自動実行だ！」
...ちょっと待て。それは**無免ドライバーをフェラーリに乗せて公道に放つ**ようなものだ。

AIの出力は、まだ安定していない。
ハルシネーション（嘘）をつくかもしれないし、文脈違いの失礼な発言をするかもしれない。
最初の1ヶ月は、必ず**「Human-in-the-loop（人間が介在するループ）」**を守れ。

### あなたの新しい役職：承認権者
ここでのあなたの役割は「ライター」ではない。
「新入社員（AI）が持ってきた原稿をチェックし、ハンコを押す部長」だ。

**【Antigravity 承認フロー】**
1.  **Draft (起案)**: AIがプロンプトに基づき、3〜5件の投稿案を作成する。
2.  **Select (選定)**: あなたがその中で「一番マシなもの」を選ぶ。
3.  **Refine (修正)**: 必要なら「てにをは」やニュアンスを微修正する。
4.  **Approve (承認)**: 実行ボタン（コマンド）を押して投稿する。

このプロセス全体で、所要時間は**3分**だ。
ゼロからネタを考えて書く場合の30分〜1時間に比べれば、10分の1のコストで済む。

## 2. 修正ルール：「30%以上直すな」

### プロンプト欠陥のシグナル
もし、AIが出してきた原稿をあなたが**30%以上**書き直しているとしたら。
それは「AIが悪い」のではない。
**「あなたのプロンプト（指示）が悪い」**のだ。

*   **書き直し率 10%以下**: 正常。語尾や固有名詞の修正のみ。
*   **書き直し率 50%以上**: 異常。AIはあなたの意図を理解していない。

書き直す前に、手を止めてプロンプトを見直そう。
「もっとフランクに書け」という指示が足りていないのか？
「専門用語を使うな」という制約が抜けているのか？
**個別の原稿を直す時間を、プロンプトを直す時間（投資）に回せ。**

### フィードバックループの構築
修正した内容は、必ずAIにフィードバックせよ。
LLMは文脈学習（In-context Learning）ができる。

**悪いフィードバック:**
「これじゃダメ。書き直して。」（理由がない）

**良いフィードバック:**
「B案の方向性はいいが悪ふざけが過ぎる。絵文字は廃止し、『〜だ』調に戻して、再度3案出せ。」

これを繰り返すことで、このチャットセッション内でのAIの精度は飛躍的に向上する。

## 3. 実践ケーススタディ：トーンの調整

実際に私が経験した「部長業務」の様子を見せよう。

**【Situation】**
ターゲットは「忙しい個人開発者」。
トピックは「深夜のデバッグの辛さ」について。

**【AIの初稿（Draft）】**
> 「深夜のデバッグって大変ですよね💦 でも、バグが直った時の喜びはひとしお！✨ 今日も一日頑張りましょう！ #エンジニア #プログラミング初心者」

**【部長（私）の心の声】**
「うわっ...まぶしい。キラキラしすぎだ。我々はそんな爽やかな気持ちでデバッグしていない。もっと殺伐としてるんだよ」

**【修正指示（Review）】**
> 「トーンが明るすぎる。我々は深夜3時にカフェイン中毒になりながら、血走った目でログを追っているんだ。
> もっとシニカルに、疲労感を漂わせろ。絵文字は全廃。ハッシュタグも不要。『喜び』ではなく『徒労感からの解放』に焦点を当てろ。」

**【AIの修正稿（Refined）】**
> 「深夜3時のデバッグは、プログラミングではない。あれは自分の過去の愚かさとの対話だ。
> 解決した瞬間に感じるのは達成感ではない。『やっと寝られる』という安堵だけだ。
> ログを閉じろ。寝ろ。」

**【部長（私）の判断】**
「よし、これだ。承認（Approve）。」

このやり取りにかかった時間は45秒だ。
このクオリティの文章をゼロからひねり出すには、深夜のテンションが必要だが、部長モードなら昼休みのコーヒー片手にできる。

## 4. この章のまとめとマイルストーン

### マイルストーン

*   [x] 修正量が30%を超えたら、原稿ではなくプロンプトを修正する癖がついた。
*   [x] 自分の好みのトーンになるよう、具体的な修正指示を出せるようになった。
*   [x] **「自分で書くより、直すほうが圧倒的に楽だ」**という事実を体感した。

次章は「守り」の話だ。
運用を続けていると、必ずAPIエラーや「何を書けばいいかわからない」というネタ切れ（AI側のスタック）が起きる。
トラブルを自力で解決するための実務フローを伝授する。
