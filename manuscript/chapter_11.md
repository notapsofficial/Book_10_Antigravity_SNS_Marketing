# 第11章：エラー・スクショ・AIと一緒に突破した話

## 1. 無情な401エラー

意気揚々と `python main.py` を実行した私の目に飛び込んできたのは、期待していた "Success" の文字ではなく、大量のトレースバック（エラーログ）でした。

```
tweepy.errors.Unauthorized: 401 Unauthorized
Timestamp: ...
Error Code: 32
Could not authenticate you.
```

「なんで！？」
お金も払った。キーもコピペした。コードもAIが書いた。
完璧なはずなのに、なぜ拒絶されるのか。

以前の私なら、ここでStack Overflowを検索し、似たような症例を探して数時間を浪費していたでしょう。
しかし、今の私にはAntigravityがいます。

## 2. エラーログこそが最高のプロンプト

私はターミナルの画面をスクリーンショットに撮り、何も考えずにAntigravityに投げつけました。
「エラーが出た。助けて」

Antigravityの回答は、秒速でした。

> 「401 Unauthorized ですね。
> コード自体は正しいですが、このエラーが出る場合、多くは **Developer Portal側の設定ミス** です。
> 先ほど確認した『User authentication settings』のところで、**OAuth 1.0a** が有効になっているか、そして **Application Permissions** が **Read and Write** になっているかをもう一度確認してください。
> 特に、キーを生成した『後』に権限を変えた場合、**キーの再生成（Regenerate）**が必要です」

「キーの再生成が必要」
……そんなこと、どこにも書いてなかったぞ（書いてあったかもしれないが見落としていた）。
私はDeveloper Portalに戻り、権限を確認しました。案の定、設定変更後にキーを作り直していませんでした。

## 3. ペアプログラミングの感覚

言われた通りにキーを再生成し、`.env` ファイルを書き換えました。
再び実行。今度は `403 Forbidden`。

「またか！」
再びスクショを貼る。「今度は403」

> 「403の場合、もしあなたがFreeプラン（無料版）を使っていたときの名残で、古いエンドポイント（V1.1）を叩いている可能性があります。
> Basic TierではV2のエンドポイントを使う必要があります。コードのこの部分を `client.create_tweet` に書き換えてください」

そうやって、エラーが出るたびにログを投げ、解説を受け、修正する。
そのプロセスは、孤独なデバッグ作業ではありませんでした。
隣に熟練の先輩エンジニアが座っていて、「あー、それはあれだよ」と指差してくれるような感覚。
まさに**「ペアプログラミング」**でした。

## 4. 壁が崩れる音

3度目の正直。
コードを修正し、保存し、深呼吸して実行コマンドを叩きます。

`python main.py`

一瞬の沈黙の後、ターミナルに静かに文字が表示されました。

```
Response: 201 Created
ID: 187515...
Text: Hello, Antigravity.
```

エラーはありません。
私は震える手で、ブラウザのXを開きました。
そこには、私の「代理人」が呟いた、最初の言葉が刻まれていました。

## 到達状態 (Check Point)

*   [ ] エラーが出てもパニックにならず、ログをAIに見せる習慣がついた。
*   [ ] 「システム側の設定ミス」と「コードのミス」の切り分けをAIに手伝ってもらった。
*   [ ] 孤独な開発ではなく、AIとの共同作業（ペアプロ）としてのデバッグを楽しめた。

**「エラーはもう行き止まりではありません。AIという相棒との会話の糸口に過ぎません」**

ついに壁を突破しました。
繋がったのです。
次章、この小さな「201 Created」が私にもたらした、とてつもない解放感について語ります。
